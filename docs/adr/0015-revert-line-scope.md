# 0015 ADR 0014 §A の動的路線発見を撤回し, scope 内で駅番号網羅性を改善する

## Status
Accepted (US-044 で採用)。ADR 0014 を supersede する。ADR 0007 (東海4県マスタ取り込み仕様) のホワイトリスト方式に戻し, ADR 0008-0013 (駅番号取り込みパイプライン) はそのまま動作する。

## Context
ADR 0014 で「4 県内に駅番号 (P296) を持つ駅が紐付く全鉄道路線を SPARQL で動的発見」する scope 拡大を採用した。再取込の結果は:

- Line 49 → 88 (+74 第三セクター/私鉄/新幹線)
- Station 818 → 1200 (+382)
- StationLine 907 → 1335 (+428)
- 駅番号付番済 link: 546 → 697 (52.2%)

しかしこれは ADR 0007 で凍結したスコープ (JR 14 路線 + OTHER_OPERATORS 5 社) を **明示的な合意なく拡張** したものであり, ユーザレビューで「あらかじめ適用する路線数は絞ったはず」「中央新幹線は不要」「ローカル路線は不要」と指摘を受けた (US-011/013 でやり取りした取り込み範囲を逸脱)。サンドボックスの想定規模 (2-3 画面の小規模システム) も超えている。

加えて, ADR 0014 の主目的の一つだった「名古屋駅の駅番号網羅性」も達成できておらず (動的取り込み後も JR名古屋駅 の駅番号は空のまま), その原因を調査したところ:

1. **Wikidata データ不備**: JR東海名古屋駅 (Q110799384) と近鉄名古屋駅 (Q1109915) は P296 = "ナコ" (電報略号, ADR 0009 で除外対象) のみで, 現代の駅番号 (CA68 等) が登録されていない。これは Wikidata 編集者の整備不足で, 我々のロジックでは救えない。
2. **ロジックの隙間**: あおなみ線名古屋駅 (Q124417968) の P296 = "AN01" (qualifier 無し, lineLink は単独 = あおなみ線 1 件) は, 本来 ADR 0010 §1 で「単独路線駅は採用」されるべきだが, ADR 0012 のマージ処理で JR名古屋駅 (Q110799384) に吸収される際に **複数 lineLink 状態になってから ADR 0010 を適用するため ambiguous でスキップ** されている。

(2) はロジック修正で救える。(1) はデータソースの限界。

## Decision

### A. 動的路線発見を撤回 (§A of ADR 0014)

`fetchAllNonJRLines` 等の動的取り込み関数は実装しない。`fetchOtherLines` (ADR 0007 §1.2 OTHER_OPERATORS ベース) のままとする。

JR_LINE_QIDS 14 路線 + OTHER_OPERATORS 5 社 (= 約 49 路線) のスコープを維持。

### B. 「単独路線駅の unattached → attached」変換をマージ前に実施

`fetchStationsForLines` の処理順を以下のように変更する:

1. SPARQL から行を取り込んで `acc: Map<Q-ID, Acc>` を構築 (現状通り)
2. **【新規】 各 Acc について, lineIds.size === 1 かつ unattachedCodes.size > 0 の場合, unattachedCodes を全て codesByLine[唯一の lineId] に移し unattachedCodes を空にする** (ADR 0010 §1 のマージ前適用)
3. ADR 0012 のマージステップで複数 Q-ID を 1 つに統合
4. ADR 0011 の prefix 学習 + ADR 0010 §2/§3 の disposition 処理

これにより, あおなみ線名古屋駅 (Q124417968) の AN01 はステップ 2 で `codesByLine[あおなみ線]={AN01}` に確定し, ステップ 3 のマージ後も保持される。マージ後の名古屋駅で「あおなみ線 = AN01」が正しく付番される。

### C. audit レポート出力 (§D of ADR 0014 を引き継ぎ)

import 終了時に `docs/audit/missing-station-codes.md` を生成する。フォーマットは ADR 0014 §D のまま:

- サマリ (Station 件数, StationLine 件数, 付番率, 未付番件数)
- 未付番リスト (駅名 / よみがな / 路線 / 種別 / 駅 Q-ID / 路線 Q-ID)

このレポートが「Wikidata データ不備で自動補完できない駅」の実体的なリストになる。手動補完はユーザが原則禁止しているが, リストの存在自体は upstream Wikidata 編集の参考データとして有用。

### テスト
- ADR 0010 §1 のマージ前適用: 単独路線駅 + unattached → 確定取り込み
- マージ後も保持されることを単体テストで検証

### 既存データ
ADR 0008 系と同じく `--clean` 再取込で補正。

## Consequences

### 利益
- ADR 0007 の凍結スコープを尊重 (取り込みは 49 路線、約 800 駅 規模に戻る)。
- あおなみ線名古屋駅 = AN01 等, マージで失われていた qualifier-無し 単独路線 code を救える。
- audit レポートで残作業が見える化される (Wikidata 編集者への upstream 修正依頼の根拠にもなる)。

### 代償・リスク
- 第三セクター鉄道の駅は再び取り込み対象外 (ADR 0007 の通り)。長良川鉄道, 愛知環状鉄道等の駅マスタは無し。
- JR名古屋駅の駅番号は依然として空 (Wikidata データ不備)。これは upstream の問題で本サンドボックスでは解消不能。
- ADR 0014 を Superseded にしたが, 再びスコープ拡大の議論が起きた場合の出発点としては残す。

### 影響範囲
- ADR 0014 を Superseded に変更 (本 ADR で代替)。
- ADR 0007 (取り込みスコープ) は引き続き active。
- ADR 0010 §1 のロジックを `fetchStationsForLines` のマージ前に適用するよう実装位置を変更 (意味は変わらない)。
- audit レポート機能は新規追加。
