# 0005 マスタデータソース選定 (東海4県の路線・駅マスタ取り込み)

## Status
Accepted (2026-05-03)

## Context

US-011 でマスタ対象を東海4県 (愛知/岐阜/三重/静岡) の鉄道路線および駅に拡張することが決まった。これに伴い 2026-05-03 時点で `backend/prisma/seed.ts` および `frontend/src/lib/lines.ts` のハンドキュレーション (東京メトロ圏 14 路線 + 12 駅) はいったん削除され, マスタは空の状態。

東海4県の規模感:
- **鉄道路線**: 約 50〜80 (JR東海/JR西日本/名鉄/近鉄/愛知環状鉄道/東海交通事業/愛知高速交通/名古屋臨海高速鉄道/名古屋市交通局/豊橋鉄道/明知鉄道/長良川鉄道/養老鉄道/伊勢鉄道/三岐鉄道/伊賀鉄道/四日市あすなろう鉄道/静岡鉄道/遠州鉄道/天竜浜名湖鉄道/大井川鐵道/岳南電車/岳南鉄道/伊豆箱根鉄道/伊豆急行/伊豆急/しずてつジャストライン *一部* など)
- **駅**: 約 1,500〜2,500
- バス停留所まで含めると一桁増える (本 ADR では対象外)

データソースの候補を評価した:

| 候補 | 取得コスト | ライセンス | 鉄道カバレッジ | バス | 接続情報 | 自動更新 |
|---|---|---|---|---|---|---|
| **国土数値情報 S05 鉄道時系列** (国交省) | 中 (Shapefile→ETL) | 政府標準利用規約 (CC-BY 相当) | 全国網羅 | × (S10 別データ) | △ (路線↔駅紐付けあり) | 年次更新 |
| **Wikidata SPARQL** (P31=鉄道駅 P81=路線) | 低 (SPARQL) | CC0 | 国内ほぼ網羅 | △ | ○ | コミュニティ依存 |
| **OpenStreetMap (Overpass)** | 中 | ODbL (継承条項) | 詳細 | ○ | ○ (route relation) | コミュニティ依存 |
| **各事業者公式 (JR東海/名鉄/近鉄等)** | 高 (個別 API/CSV) | 規約バラバラ | 100% (公式) | △ | ○ | 事業者次第 |
| **静的キュレーション** | 高 (人力) | — | 限定的 | — | 手動 | 不可 |

## Decision

**Wikidata SPARQL を一次ソースとして採用する。** 理由は以下:

1. **ライセンス**: CC0 で再配布制約が無く, サンドボックスの公開リポジトリでも安心して扱える。ODbL (OSM) と違い「派生データの公開義務」が無い。
2. **取得コスト**: SPARQL クエリ 1 本で「県 (P131 = 愛知県/岐阜県/三重県/静岡県) に所在する駅 (P31 = Q12819564 鉄道駅, Q55488 駅) と接続路線 (P81)」を JSON で取得できる。Shapefile ETL を組む必要がない。
3. **接続情報**: 路線-駅の M:N が P81 (所在路線) で表現されており, 現スキーマの `StationLine` にそのままマップできる。
4. **更新性**: 新駅開業はコミュニティが比較的早く反映する。完璧ではないが, 業務クリティカルでない本サンドボックスには十分。

代替案を採用しない理由:
- 国土数値情報: Shapefile ETL が重く, サンドボックスの趣旨 (要件→設計→製造→テスト の追体験) から逸脱する。
- OSM: ODbL の継承条項解釈が, 派生 DB を本 GitHub repo に含める形で問題が出る可能性がある (法務判断が要る)。
- 事業者公式: 各社で API/CSV 規約が異なり, 統合に時間がかかる。
- 静的キュレーション: 1,500+ 駅の手書きは現実的ではない。

### 取り込み方式

**one-shot バッチによる初回取り込み + 必要時の手動再実行**, とする。定期同期 (cron 等) は本サンドボックスでは実装しない (理由: 運用負荷不要 / 工程追体験という主目的への影響なし)。

実装方法:

1. `backend/scripts/import-master-tokai.ts` を新設
   - Wikidata SPARQL endpoint (`https://query.wikidata.org/sparql`) に対してクエリ送信
   - 結果 JSON を Line / Station / StationLine の Prisma モデルにマップ
   - upsert で冪等に書き込み (id は Wikidata Q-ID をそのまま使う)
2. `package.json` に `pnpm --filter backend run import:master:tokai` スクリプトを追加
3. CI では実行しない (テストはマスタを必要に応じて inline fixture で生成する方針, ADR 0006 参照)
4. データソース URI と取得日時を `Line.operator` 等とは別の `Line.sourceUri` / `importedAt` 列で保持できるよう, 後続マイグレーションで列を足す (要 ADR 改訂)。本 ADR では Wikidata 取り込み採用の確定までにとどめ, 列追加は US-011 実装フェーズの ADR 改訂で確定させる。

### Line.id / Station.id の決め方

現行スキーマでは Line.id はスラッグ (`jr-yamanote` 等), Station.id は `cuid()` 自動採番。これを以下に改める:

- `Line.id`: Wikidata Q-ID (例: `Q1129155` JR山手線) を文字列でそのまま採用
- `Station.id`: Wikidata Q-ID をそのまま採用 (現状 `cuid()` だが Q-ID は string なので互換)

これにより Wikidata 側の更新を id ベースで突合できる。

### `kind` の付与

Wikidata 側に `kind` (train/subway/bus/other) と直接対応する property は存在しないため, 以下のヒューリスティクスで導出する:

- 運営会社名に「地下鉄」「Metro」が含まれる → `subway`
- 路線種別 (P1029) が「鉄道路線」かつ subway 条件に該当しない → `train`
- それ以外 → `other`

バスは本 ADR の対象外 (将来別 ADR で追加検討)。

### バスの扱い

US-011 の対象外とする。鉄道のみで実装を進める。バスを将来追加する場合は GTFS-JP もしくは事業者公式の取り込みを別 ADR で扱う。`LineKind` enum には `bus` が残るが, マスタ件数 = 0 のままで運用する。

### 静岡県の特殊事情

静岡県は東日本/中部の境界域で JR東海/JR東日本両方の駅がある (例: 熱海以西は JR東海, 熱海以東は JR東日本)。県境ではなく **県 = 静岡県** で抽出するため, 熱海駅以西のみが対象となる (熱海駅自体は静岡県, 含まれる)。利用者が誤解しないよう, 取り込み完了通知に「JR管轄ではなく所在県基準で取得」と明記する。

## Consequences

利点:
- CC0 ライセンスにより配布・再利用が自由。
- SPARQL の単一クエリで 4 県分を一括取得でき, 取り込みスクリプトが極端に短く済む見込み (100行程度)。
- 路線-駅 M:N がそのままマップでき, スキーマ変更が不要。
- id を Wikidata Q-ID にすることで, 将来再取り込み時の差分計算が容易。

代償・リスク:
- Wikidata の正確性はコミュニティ依存。新駅開業や路線変更が即時反映されないことがある。これを補正するために US-012/013 (画面 CRUD) が必要 (ADR 0006)。
- SPARQL クエリ実行は外部サービスへの依存。Wikidata 側のレート制限や障害時に取り込みが詰まる可能性。これは初回取り込みの一過性なのでリトライで吸収可能。
- `kind` のヒューリスティクスは精度 100% にならない。実態と乖離した分は US-012 (路線編集) で運用補正する前提。
- バスを対象外にしたため, バスを使う通勤経路は引き続き「種別=その他 + 路線=未選択」での自由入力に依存する (UI 上は LineKind enum に `bus` を残しつつマスタは 0 件のまま, とする)。
- `Line.id` をスラッグから Q-ID に変える破壊的変更が発生する。マスタが空の今のうちに切り替えるためデータ移行は不要だが, 既存の SQL/コードでハードコード参照が残らないよう US-011 実装時に総点検する。
- Wikidata の更新を取り込みたくなった際の「差分取り込み戦略」は本 ADR では未確定。再取り込み時に既存路線/駅のマージ方針を別途決める必要がある。

将来検討余地:
- バスマスタの取り込み (GTFS-JP)
- 取り込みデータの出典明示 (UI フッタ等への Wikidata 出典クレジット)
- 全国スコープへの拡張 (現在は東海4県のみ。将来全国に広げる場合, データ量は数万件規模となり SQLite 継続可否を再検討する)
