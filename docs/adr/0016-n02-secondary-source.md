# 0016 国土数値情報 N02 を駅マスタの従データソースとして併用する

## Status
Accepted (US-045 で採用)。ADR 0007 (東海4県マスタ取り込み仕様) を拡張する。ADR 0008-0013, 0015 (駅番号取り込みパイプライン) はそのまま動作する。

## Context
ADR 0007 + 0015 で取り込みスコープを「JR 14 路線 + OTHER_OPERATORS 5 社」に絞り, Wikidata を単一データソースとしている。US-044 完了時点の状態:

- Station: 818
- StationLine: 907
- 駅番号付番済: 561 (61.9%)
- 未付番: 345 (主に Wikidata 側のデータ不備)

未付番駅の主因は Wikidata の P296 statement 未登録 (例: JR東海名古屋駅は P296=`ナコ` のみで CA68 等が欠落)。これは upstream の問題で本サンドボックスでは解消できない。

ユーザレビュー (US-045) で「他のデータソースを併用しないか」という指摘を受けた。

代替/補完候補を比較:

| ソース | 駅情報 | 路線情報 | 駅番号 | 座標 | 形式 | ライセンス |
|---|---|---|---|---|---|---|
| **Wikidata** (現主) | ○ | ○ | △ (qualifier 整備不足) | △ | RDF/SPARQL | CC0 |
| **国土数値情報 N02** | ◎ | ◎ | × **含まれない** | ◎ | GeoJSON/Shapefile | 商用可 |
| **Wikipedia 駅ナンバリングページ** | × | × | ◎ | × | HTML | CC-BY-SA |
| **OpenStreetMap (Overpass)** | ○ | ○ | ○ (ref タグ) | ○ | XML | ODbL |
| **各事業者公式サイト** | ◎ | ◎ | ◎ | ○ | バラバラ | 個別確認 |

N02 は **駅番号を含まない** ため, N02 を主従の主に切り替えると駅番号が空になる。Wikidata の駅番号情報を活かすには Wikidata 主の構成が望ましい。

一方 N02 は **駅・路線の網羅性と座標品質** で Wikidata を凌駕する (政府公式・年次更新)。これらを補完源として併用すれば, US-043 で問題になった「JR以外の路線取りこぼし」「座標欠落駅」もカバーできる。

## Decision

### A. 役割分担

- **主**: Wikidata (現状維持)
  - 駅番号 (P296) と qualifier (P81/P518)
  - 路線階層エイリアス (Q1190152 など)
  - kana (P1814)
  - 名前正規化 (NAME_PREFIX_PATTERNS)
- **従**: 国土数値情報 N02 (新規)
  - 路線・駅・座標の網羅性補完
  - 運営会社・正式路線名の補完

### B. データ取得

国土数値情報ダウンロードサービス (https://nlftp.mlit.go.jp/ksj/) から N02 (鉄道) GeoJSON を 4 県分ダウンロードする。

- 都道府県別配布なので 4 県 (愛知/岐阜/三重/静岡) を個別取得
- バージョンは最新年度 (取り込み時の固定バージョンで再現性確保)
- ダウンロード結果は `backend/scripts/data/n02/<pref>.geojson` にキャッシュ (.gitignore で除外, 取り込み時に未取得なら自動 fetch)
- アクセス頻度低 (年次データなので頻繁な fetch は不要)

### C. データ突合 (merge ロジック)

Wikidata 取り込み完了後 (= ADR 0012 のマージ + ADR 0011 の prefix routing 適用後) に N02 取り込みを実施。

各 N02 駅について:
1. Wikidata 由来 Station と **座標距離 < 200m + 名称一致** (正規化後比較) で突合
2. 一致する Station が見つかった場合:
   - 路線: 不足している lineLink を追加 (ただし駅番号は付与しない — Wikidata の駅番号情報を上書きしないため)
   - 運営会社・正式路線名: Wikidata 値が空なら N02 値で埋める
3. 一致する Station が見つからない場合:
   - 新規 Station を作成 (Station.id は cuid, sourceUri は N02 のレイヤ識別子)
   - 駅番号は空のまま (N02 にデータなし)

突合距離の閾値 (200m) は ADR 0012 のマージ閾値 (500m) より厳しく設定。N02 と Wikidata で座標表記が微妙にずれるが 200m あれば実用的に十分。

### D. Line マスタの扱い

N02 の路線情報も Wikidata 由来 Line と突合する:
1. Line.name (路線名) が一致 → Wikidata Line にマップ
2. 一致しない (Wikidata に無い路線) → Line を新規追加 (Line.id は path-safe な slug, sourceUri は N02)

ただし当面は **JR + OTHER_OPERATORS スコープの路線のみ** 取り込み対象とする。N02 で発見した「ホワイトリスト外路線」(第三セクター等) は **取り込まない** (ADR 0007 のスコープ尊重)。

### E. テスト

- N02 GeoJSON を `__fixtures__/n02-aichi-mock.json` に縮小サンプル化して unit test
- 「座標一致 + 名称一致でマージされる」「閾値外の駅は別 Station として残る」「新規駅の sourceUri が N02 を指す」等を検証

### F. audit レポート (継続)

ADR 0015 §C の `docs/audit/missing-station-codes.md` は引き続き出力。N02 補完で新規取り込みされた駅も載る (駅番号は N02 にないので空のまま, 手動補完候補として明示)。

## Consequences

### 利益
- Wikidata に無い駅 (第三セクター駅等は引き続き対象外だが, JR + OTHER_OPERATORS 範囲で取りこぼされていた駅) を補完。
- 座標品質が向上 (N02 は GIS 品質)。
- 運営会社・正式路線名が公式表記で揃う。

### 代償・リスク
- N02 の取得処理 (HTTP fetch + GeoJSON parse) を追加する分のメンテ負荷。
- 国土数値情報のデータ更新は年次なので, 古いデータで取り込んでしまうリスク。バージョンを明示的に固定する。
- 名称正規化の差分: Wikidata と N02 で「東海道本線 (名古屋地区)」と「JR東海道本線」のように表記が異なる場合がある。突合ロジックで正規化する。
- 住所/郵便番号からの最寄り駅算出 (US-046 系想定) のためには座標が必須なので, 座標品質向上は将来基盤として有用 (が, US-045 では実装しない)。

### 影響範囲
- ADR 0007 §1 (取り込み source 規定): 「Wikidata SPARQL のみ」→「Wikidata SPARQL + 国土数値情報 N02」に拡張。
- ADR 0008-0013, 0015 はそのまま動作 (本 ADR は merge 後の post-step として動作する)。
- DB の Station/StationLine 件数が微増する見込み (Wikidata で取りこぼしていた駅が追加されるため)。
- 駅マスタ管理 (S10) のフィルタ UI (US-031/032) は影響なし。
