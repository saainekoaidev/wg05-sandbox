## 5. 画面遷移

| イベント | 遷移先 | 条件 |
| --- | --- | --- |
| 一般ユーザがアクセス | 本画面 (403 表示) | role !== 'admin' |
| 未認証アクセス | `/login` | セッション無し |
| 作成 / 更新成功 | `/admin/operators` (通知バナー付き) | API 200/201 |
| 削除成功 | `/admin/operators` (リスト reload + 通知バナー) | API 204 |
| 削除失敗 (参照 Line あり) | エラーバナー表示 | API 409 (referenced) |

## 6. バリデーション仕様

### 6.1 クライアントサイド
- ID (作成時): 必須 + slug 形式 (`^[a-z0-9-]+$`) + 80文字以内
- 表示名: 必須 + 80文字以内
- aliases: 任意 / 各エイリアスを `,` 区切り (内部で JSON 配列に変換して保存)
- 「作成する/更新する」押下時に検証。NG ならフォーム内バナー表示, API 呼出は行わない

### 6.2 サーバサイド
- zod (`OperatorCreateInput` / `OperatorUpdateInput`) で同等チェック + 認証 + role=admin 検証
- 失敗時 400 (validation_failed) / 401 / 403 / 404 / 409 (duplicate / referenced)

### 6.3 エラーメッセージ
| 条件 | メッセージ |
| --- | --- |
| ID 未入力 | IDを入力してください |
| ID 形式違反 | IDは半角英小文字 + 数字 + ハイフン のみ使用できます |
| 表示名 未入力 | 表示名を入力してください |
| 401 | (画面外で /login へリダイレクト) |
| 403 | 管理者権限が必要です |
| 409 (duplicate) | 同じIDまたは表示名が既に登録されています |
| 409 (referenced) | この事業者は N 件の路線で参照されているため削除できません |
| 5xx / 通信 | 保存に失敗しました。再度お試しください |

## 7. その他考慮事項

- Operator は Line / Station 双方から参照される。削除時は両方の参照件数をチェックし, 0 件以外は削除不可 (409 referenced)
- aliases は表記揺らぎ補正用 (例: Wikipedia の「東海旅客鉄道」を「JR東海」と紐付ける). Phase 1 では取り込み側はまだ aliases を読み込まない (Phase 2 以降で活用)
- Phase 2 (将来): `Line.operator` 文字列の削除 / 経路登録 (S04) で operator を明示する UI / aliases ベースの自動正規化
