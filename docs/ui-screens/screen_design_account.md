| --- | --- |

## 3. 入力項目定義

### 3.1 プロフィールセクション (US-008)

| No | 項目名 | 表示ラベル | 入力形式 | 必須 | 桁数 | バリデーション | 備考 |
| --- | --- | --- | --- | :-: | --- | --- | --- |
| 1 | メールアドレス | メールアドレス | テキスト (read-only) | — | — | — | 表示専用。本サンドボックスでは変更不可 (ADR 0004) |
| 2 | お名前 | お名前 | テキスト | ◯ | 1〜50文字 (前後空白 trim 後) | 空白のみは不可 | サインアップ時の name と同じ制約 |
| 3 | 郵便番号 | 郵便番号 | テキスト | × | 0 または 7文字 | `^\d{7}$` (ハイフン無し) | 任意項目。空欄保存で NULL に倒す |

### 3.2 パスワード変更セクション (US-009)

| No | 項目名 | 表示ラベル | 入力形式 | 必須 | 桁数 | バリデーション | 備考 |
| --- | --- | --- | --- | :-: | --- | --- | --- |
| 4 | 現在のパスワード | 現在のパスワード | パスワード (マスク) | ◯ | 8〜32文字 | 形式チェックのみ | better-auth の検証は API 側で行う |
| 5 | 新しいパスワード | 新しいパスワード | パスワード (マスク) | ◯ | 8〜32文字 | 半角英数記号、英大文字・小文字・数字を各1文字以上含む。現在パスワードと異なること | US-001 (Register.tsx) と同じ強度ルール |
| 6 | 新しいパスワード (確認) | 新しいパスワード (確認) | パスワード (マスク) | ◯ | 8〜32文字 | No.5 と一致すること | — |

各パスワード入力欄には表示/非表示トグル (「表示」「隠す」) を併設する (1 つのトグルで 3 欄をまとめて切替)。

## 4. ボタン定義

| No | ボタン名 | 表示ラベル | 動作 |
| --- | --- | --- | --- |
| 1 | 保存ボタン | 保存する | 入力内容を `PUT /api/users/me` に送信。成功時, 通知バナー「プロフィールを更新しました」を表示 |
| 2 | 戻すボタン | 変更前に戻す | 取得時の値に入力を巻き戻す。差分が無い場合は何もしない (確認ダイアログ非表示) |
| 3 | パスワード変更ボタン | パスワードを変更する | better-auth `changePassword({ currentPassword, newPassword, revokeOtherSessions: true })` を呼出。成功時、入力をクリアし通知バナーを表示 |
| 4 | 表示トグル | 表示 / 隠す | 3 つのパスワード入力欄をまとめて type=text/password に切替 (現在/新規/確認) |
| 5 | アカウント削除ボタン | アカウントを削除する | パスワード未入力チェック後 `window.confirm` で意思確認 → better-auth `deleteUser({ password })` を呼出。成功時 `/login` へ遷移 |
| 6 | 一覧戻るリンク | 経路一覧に戻る | `/routes` へ遷移 (確認ダイアログなし。差分破棄を伴う場合の警告は将来検討) |

## 5. 画面遷移

| イベント | 遷移先 | 条件 |
| --- | --- | --- |
| 保存成功 | 本画面 (バナー表示) | API 200 |
| 保存失敗 (401) | `/login` | セッション失効 |
| 保存失敗 (400/500/通信) | 本画面 (バナー表示) | エラー時 |
| 「経路一覧に戻る」押下 | `/routes` | 必ず |
| アカウント削除成功 | `/login` | better-auth deleteUser 200 (セッション破棄済み) |
| アカウント削除失敗 (パス誤り/通信) | 本画面 (バナー表示) | error.code に応じて文言切替 |
| 未認証アクセス | `/login` | セッション無し |

## 6. バリデーション仕様

### 6.1 クライアントサイド

- フォーカスアウト時 + 「保存する」押下時に実行
- 氏名: 空白のみ → 「氏名を入力してください」
- 氏名: 51文字以上 → 「氏名は50文字以内で入力してください」
- 郵便番号: 半角数字以外を含む / 7桁未満 → 「郵便番号は半角数字7桁で入力してください (ハイフン不要)」
- 郵便番号入力中はキー入力フィルタ (`replace(/\D/g, '').slice(0, 7)`) で常に半角数字 7 桁以下に正規化

### 6.2 サーバサイド

- zod (`ProfileInput`) で同等チェック。`name` は `min(1).max(50).refine(空白のみ拒否)`。`postalCode` は `string.regex(/^\d{7}$/) | "" | null` の union
- 失敗時 400 + `{ error: 'validation_failed' }`
- 成功時 200 + 更新後オブジェクト (`id`, `email`, `name`, `postalCode`)

### 6.3 エラーメッセージ (バナー)

| 条件 | メッセージ |
| --- | --- |
| GET 失敗 (5xx / 通信) | プロフィールの取得に失敗しました。再読み込みをお試しください |
| PUT 400 | 入力内容に誤りがあります。再度ご確認ください |
| PUT 5xx / 通信 | プロフィールの更新に失敗しました。再度お試しください |
| changePassword INVALID_PASSWORD | 現在のパスワードが正しくありません |
| changePassword その他失敗 / 通信エラー | パスワードの変更に失敗しました。時間をおいて再度お試しください |
| deleteUser INVALID_PASSWORD | 現在のパスワードが正しくありません |
| deleteUser その他失敗 / 通信エラー | アカウントの削除に失敗しました。時間をおいて再度お試しください |

### 6.4 パスワードフィールド単位エラー

| 項目 | 条件 | メッセージ |
| --- | --- | --- |
| 現在のパスワード | 未入力 | 現在のパスワードを入力してください |
| 新しいパスワード | 未入力 | パスワードを入力してください |
| 新しいパスワード | 桁数不足/超過 | パスワードは8〜32文字で入力してください |
| 新しいパスワード | 文字種不足 | パスワードは英大文字・小文字・数字をそれぞれ1文字以上含めてください |
| 新しいパスワード | 現パスと同一 | 新しいパスワードは現在のパスワードと異なるものにしてください |
| 確認用 | 未入力 | 確認用パスワードを入力してください |
| 確認用 | 不一致 | パスワードが一致しません |

## 7. その他考慮事項

- 保存処理中は「保存する」「変更前に戻す」を `disabled` で二重送信防止
- パスワード変更処理中は「パスワードを変更する」と表示トグルを `disabled` で二重送信防止
- 郵便番号 7 桁が満たされた瞬間にハイフン入りプレビュー (`150-0001`) を hint 行に表示 (DB 保存はハイフン無し)
- メールアドレス変更は対象外。input は `readonly` で常に hint「メールアドレスは本サンドボックスでは変更対象外です」を併記
- パスワード変更成功後は **現在のセッションは保持** され, **他デバイスのセッションのみ無効化** される (better-auth `revokeOtherSessions: true`, ADR 0004)
- パスワードバナー / プロフィールバナーは独立 (片方の操作でもう片方の通知は消えない)
- アカウント削除は **物理削除** (Session/Account/Route/RouteSegment は cascade, Verification は better-auth `afterDelete` フックで identifier=email を明示削除) (ADR 0004 §(b))
- 削除確認ダイアログの文言: 「アカウントを削除します。登録済みの経路を含む全データが完全に削除され、復元できません。本当に削除してよろしいですか?」
- 本画面への遷移動線は経路一覧 (`/routes`) のフッタ「ユーザー: foo@example.com」リンクから (US-008 で結線済み)
