## 2. 画面レイアウト

```
┌──────────────────────────────────────────────────────────────┐
│  Admin / Lines                          [ + 新規作成 ]         │
│  路線マスタ管理                                                │
│  登録済みの路線を確認・追加・編集・削除できます (管理者専用)     │
├──────────────────────────────────────────────────────────────┤
│  [通知バナー / エラーバナー (任意)]                              │
│                                                              │
│  ┌────┬──────────┬──────┬──────────┬──────┬──────┬──────────┐│
│  │ ID │ 路線名     │ 種別  │ 運営会社  │ 参照 │ 接続駅│ 操作     ││
│  ├────┼──────────┼──────┼──────────┼──────┼──────┼──────────┤│
│  │ jr-tokaido │ JR東海道線 │ 電車 │ JR東海 │  0  │  5  │[編集][削除]││
│  │ metro-meijo│ 名城線      │ 地下鉄│ 市交   │  2  │ 12  │[編集][削除]││ ← 参照>0 で削除 disabled
│  └────────────────────────────────────────────────────────────┘│
│                                                              │
│  ─── New Line / Edit Line (フォーム) ───                       │
│   ID *  [____________________]   ※編集時は disabled         │
│   路線名 * [____________________]                            │
│   種別 * [電車 ▼]                                            │
│   運営会社 [____________________]                            │
│                                                              │
│   [ 作成する / 更新する ]   [ キャンセル ]                     │
├──────────────────────────────────────────────────────────────┤
│  アカウント設定                            経路一覧へ            │
└──────────────────────────────────────────────────────────────┘
```

「+ 新規作成」「編集」は **専用画面に遷移する Link** (US-025)。新規は `/admin/lines/new`, 編集は `/admin/lines/:id/edit`。フォーム送信成功後は `/admin/lines` に navigate state.notice 付きで戻り、一覧に通知バナーが表示される。

US-031: 一覧上部に「種別」フィルタ (`すべて / 電車 / 地下鉄 / バス / その他`) を設置する。クライアント側の filter で表示行を絞り込む (件数が多くないので server filter は導入しない)。

US-034: フィルタの選択値は sessionStorage (key: `admin-lines-filter`) に保存し, `/admin/lines/new` や `/admin/lines/:id/edit` から戻ってきたときに復元する。画面 mount 時に再 fetch を行うため復元と同時に最新状態が表示される。

## 4. ボタン定義

| No | ボタン名 | 表示ラベル | 動作 |
| --- | --- | --- | --- |
| 1 | 新規作成 (ヘッダ / 空状態) | + 新規作成 | フォームを `create` モードで開き入力欄をクリア |

## 5. 画面遷移

| イベント | 遷移先 | 条件 |
| --- | --- | --- |
| 一般ユーザがアクセス | 本画面 (403 表示) | role !== 'admin'。「経路一覧に戻る」リンクのみ提示 |
| 未認証アクセス | `/login` | セッション無し |
| GET /api/users/me 401 | `/login` | セッション失効 |
| 作成 / 更新成功 | 本画面 (フォーム閉じる + 通知バナー) | API 200/201 |
| 削除成功 | 本画面 (リスト reload + 通知バナー) | API 204 |
| 削除 409 (参照中) | 本画面 (バナーに件数表示) | RouteSegment が参照中 |

## 6. バリデーション仕様

### 6.1 クライアントサイド

- ID (作成時のみ): 未入力 / 形式違反 / 80文字超過
- 路線名: 未入力 / 80文字超過
- 運営会社: 80文字超過
- 「作成する/更新する」押下時に実行。NG ならフォーム内バナーで通知し API 呼出は行わない

### 6.2 サーバサイド

- zod (`LineInput`) で同等チェック + 認証 + role=admin 検証
- 失敗時 400 (validation_failed) / 401 (unauthorized) / 403 (forbidden) / 404 (not_found) / 409 (duplicate or in_use)

### 6.3 エラーメッセージ (フォームバナー)

| 条件 | メッセージ |
| --- | --- |
| ID 未入力 | IDを入力してください |
| ID 形式違反 | IDは半角英数字 + ハイフン/ドット/アンダースコアのみ使用できます |
| 路線名未入力 | 路線名を入力してください |
| 80文字超過 (各項目) | 〜は80文字以内で入力してください |
| 401 | (画面外で /login へリダイレクト) |
| 403 | 管理者権限が必要です |
| 400 | 入力内容に誤りがあります |
| 404 | 編集対象の路線が見つかりませんでした (削除済み?) |
| 409 (重複) | 同じIDまたは路線名が既に登録されています |
| 409 (in_use) | この路線は ◯ 件の経路で使用中のため削除できません。先に対象経路を編集してください |
| 5xx / 通信 | 保存に失敗しました。再度お試しください / 路線の削除に失敗しました。再度お試しください |

## 7. その他考慮事項

- 一覧は `kind` 昇順 + `name` 昇順でグルーピング表示 (subway → train → bus → other)
- 削除ボタンは `routeSegmentCount > 0` のとき disabled + `title` ツールチップで理由を表示
- 並び替え/ページング/検索は本 US では未実装 (件数が増えた段階で別 US で検討)
- フォームはインライン展開で複数同時編集はできない (1 つのフォーム状態を共有)
- 管理者昇格は CLI のみ (`pnpm --filter backend exec tsx scripts/grant-admin.ts <email>`)。画面からの権限変更 UI は提供しない (ADR 0006 §1)
