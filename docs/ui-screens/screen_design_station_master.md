| --- | --- |

## 3. 入力・表示項目定義

### 3.1 入力項目（検索条件）

| No | 項目名 | 表示ラベル | 入力形式 | 必須 | 桁数 | バリデーション | 備考 |
| --- | --- | --- | --- | :-: | --- | --- | --- |
| 1 | キーワード | 駅名 / よみがな | テキスト | △ | 最大30文字 | 部分一致（前方優先） | ひらがな・カタカナ・漢字・ローマ字いずれも可 |
| 2 | 種別 | 種別 | セレクトボックス | △ | - | `train` / `subway` / `bus` / `other` または `すべて` | 既定値「すべて」 |
| 3 | 路線 | 路線 | セレクトボックス | △ | - | 路線マスタから選択 | 既定値「すべての路線」 |

> 検索条件はキーワード / 種別 / 路線のいずれかが指定されていれば検索可。すべて未指定時は「条件を入力してください」を表示。

### 3.2 表示項目（検索結果）

| No | 項目名 | 表示ラベル | 種別 | 桁数 | 備考 |
| --- | --- | --- | --- | --- | --- |
| 1 | No | # | 整数 | - | 1始まりの連番 |
| 2 | 種別 | 種別 | 文字列 (enum) | - | 色分けタグ表示（`tag-train` / `tag-subway` / `tag-bus` / `tag-other`） |
| 3 | 駅名 / 停留所 | 駅名 / 停留所 | テキスト | 最大50文字 | バス停留所の場合も同カラムに表示 |
| 4 | よみがな | よみがな | テキスト | 最大50文字 | ひらがな |
| 5 | 主要路線 | 主要路線 | テキスト | 最大100文字 | カンマ区切りで主要路線を列挙 |
| 6 | 選択 | 選択 | ボタン | - | この駅を呼出元フォームに返す |

### 3.3 Station マスタとの対応

サーバ側 `GET /api/stations` のレスポンス各要素の想定キー:

| キー | 型 | 必須 | 備考 |
| --- | --- | :-: | --- |
| `id` | `string` | ◯ | 駅マスタID（cuid） |
| `kind` | `"train" \| "subway" \| "bus" \| "other"` | ◯ | 種別 |
| `name` | `string` | ◯ | 駅名 / 停留所名（最大50文字） |
| `kana` | `string` | ◯ | よみがな（最大50文字） |
| `lines` | `string[]` | ◯ | 主要路線名の配列（表示時はカンマ連結） |

## 4. ボタン定義

| No | ボタン名 | 表示ラベル | 動作 |
| --- | --- | --- | --- |
| 1 | 検索ボタン | 検索 | 入力された条件で `GET /api/stations?q=...&kind=...&line=...` を呼出。結果を表に反映 |
| 2 | クリアボタン | クリア | 検索条件をリセットし、結果も初期状態に戻す |
| 3 | 選択ボタン | 選択 | 行の `name` を `postMessage({ type: 'station-pick', name })` で呼出元に返し、ポップアップを閉じる（単独画面の場合は経路登録画面 ([screen_design_route_register.md](screen_design_route_register.md)) もしくは経路編集画面 ([screen_design_route_edit.md](screen_design_route_edit.md)) に戻る） |
| 4 | 閉じるボタン | 閉じる | `window.close()` を呼出（選択なし）。単独画面の場合は呼出元へ戻る |

### 4.1 ポップアップ返却仕様（postMessage）

```js
// 駅マスタ参照ポップアップ側 (本画面) の実装イメージ
function pickStation(name) {
  if (window.opener && !window.opener.closed) {
    window.opener.postMessage({ type: 'station-pick', name }, '*');
  }
  window.close();
}
```

- `event.data.type === 'station-pick'` を識別子とする
- 親側 ([screen_design_route_register.md](screen_design_route_register.md) / [screen_design_route_edit.md](screen_design_route_edit.md)) は `data-target` で対象入力欄を覚えておき、駅名を反映する
- `targetOrigin` は本来は親画面のオリジンを指定すべきだが、開発時は `*` を許容（実装時に固定オリジンへ変更）

| イベント | 遷移先 | 条件 |
| --- | --- | --- |

## 6. バリデーション仕様

### 6.1 クライアントサイド

- キーワードは1文字以上で検索可
- キーワード・種別・路線のすべてが未指定で検索ボタン押下時は警告
- 結果が0件の場合は表本体を `.empty` で空状態メッセージに置換

### 6.2 サーバサイド

- `GET /api/stations` は認証必須
- クエリパラメータ `q` / `kind` / `line` は zod でバリデーション（`kind` は enum）
- SQL injection 等は Prisma ORM のパラメータバインドで吸収

### 6.3 エラーメッセージ

| 項目 | 条件 | メッセージ |
| --- | --- | --- |
| 検索条件 | キーワード・種別・路線がすべて未指定 | 駅名・種別・路線のいずれかを入力または選択してください |
| 種別 | 不正な値（手動URL改竄など） | 種別の値が不正です |
| 検索結果 | 0件 | 該当する駅が見つかりませんでした。条件を変えてお試しください |
| 検索結果 | 上限超過（51件以上） | 候補が多すぎます。条件を絞り込んでください（先頭50件のみ表示） |
| 通信 | 取得失敗 | 駅マスタの取得に失敗しました。再読み込みをお試しください |

## 7. その他考慮事項

- 駅マスタはサーバ側で固定データを保持（初期は手動投入。将来 CSV インポートで拡張）
- 検索結果は最大50件まで返却し、それ以上は条件絞り込みを促す
- ポップアップ形式と単独画面形式の両用パターンを想定（実装時にUI状況を見てどちらかを採用）
- ポップアップは `window.open(url, 'station-picker', 'width=960,height=720,resizable,scrollbars')` で開かれることを想定
- ポップアップ → 親画面への返却は `postMessage` を使用（`{ type: 'station-pick', name }`）
- キーボード操作: `↑↓` で行選択、`Enter` で確定、`Esc` で閉じる
- フリー入力経路（駅マスタに無い駅）の登録は経路登録画面側で許容しているため、本画面を経由しない入力経路も存在する
- 検索のデバウンス（入力中の自動検索）は次フェーズで検討
- 種別タグの色分けは経路一覧 / 経路詳細と同一規則（`tag-train` / `tag-subway` / `tag-bus` / `tag-other`）
