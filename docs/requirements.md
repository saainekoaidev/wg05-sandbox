# 要件定義

US-001	ユーザとして, メールアドレスとパスワードでアカウントを新規登録したい。なぜなら自分専用に通勤経路を保存したいから。
US-002	ユーザとして, メールアドレスとパスワードでログインしたい。なぜなら登録済みの経路を確認･編集したいから。
US-003	ユーザとして, 出発駅と到着駅と経由駅と区間運賃を入力して通勤経路を登録したい。なぜなら毎月の交通費を可視化したいから。
US-004	ユーザとして, 自分が登録した通勤経路を一覧で見たい。なぜなら登録済みの経路をひと目で把握したいから。
US-005	ユーザとして, 1経路の詳細情報を見たい。なぜなら経由や区間ごとの内訳を確認したいから。
US-006	ユーザとして, 既存の通勤経路を編集したい。なぜなら住所変更等で経路が変わった場合に更新したいから。
US-007	ユーザとして, 不要になった通勤経路を削除したい。なぜなら過去の不要データを整理したいから。
US-008	ユーザとして, 自分の登録済みプロフィール (氏名/郵便番号) を変更したい。なぜなら引っ越しや改姓などのライフイベントで情報が変わるから。
US-009	ユーザとして, 自分のパスワードを変更したい。なぜなら定期的なパスワード更改や情報漏洩懸念時に自衛したいから。
US-010	ユーザとして, 自分のアカウントを削除したい。なぜならサービスを利用しなくなった際に登録情報を残したくないから。
US-011	システムとして, 東海4県 (愛知/岐阜/三重/静岡) の鉄道路線および駅マスタを取り込んでおきたい。なぜなら本サンドボックスが想定する利用エリアの経路を網羅的に登録できるようにするため。
US-012	管理者として, 路線マスタを画面から登録/修正/削除したい。なぜなら US-011 の取り込み後に実態と乖離した路線情報を運用で補正したいから。
US-013	管理者として, 駅マスタおよび駅↔路線の接続情報を画面から登録/修正/削除したい。なぜなら新駅開業/廃駅/接続変更を反映したいから。
