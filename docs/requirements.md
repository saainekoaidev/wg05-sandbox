# 要件定義

US-001	ユーザとして, メールアドレスとパスワードでアカウントを新規登録したい。なぜなら自分専用に通勤経路を保存したいから。
US-002	ユーザとして, メールアドレスとパスワードでログインしたい。なぜなら登録済みの経路を確認･編集したいから。
US-003	ユーザとして, 出発駅と到着駅と経由駅と区間運賃を入力して通勤経路を登録したい。なぜなら毎月の交通費を可視化したいから。
US-004	ユーザとして, 自分が登録した通勤経路を一覧で見たい。なぜなら登録済みの経路をひと目で把握したいから。
US-005	ユーザとして, 1経路の詳細情報を見たい。なぜなら経由や区間ごとの内訳を確認したいから。
US-006	ユーザとして, 既存の通勤経路を編集したい。なぜなら住所変更等で経路が変わった場合に更新したいから。
US-007	ユーザとして, 不要になった通勤経路を削除したい。なぜなら過去の不要データを整理したいから。
US-008	ユーザとして, 自分の登録済みプロフィール (氏名/郵便番号) を変更したい。なぜなら引っ越しや改姓などのライフイベントで情報が変わるから。
US-009	ユーザとして, 自分のパスワードを変更したい。なぜなら定期的なパスワード更改や情報漏洩懸念時に自衛したいから。
US-010	ユーザとして, 自分のアカウントを削除したい。なぜならサービスを利用しなくなった際に登録情報を残したくないから。
US-011	システムとして, 東海4県 (愛知/岐阜/三重/静岡) の鉄道路線および駅マスタを取り込んでおきたい。なぜなら本サンドボックスが想定する利用エリアの経路を網羅的に登録できるようにするため。
US-012	管理者として, 路線マスタを画面から登録/修正/削除したい。なぜなら US-011 の取り込み後に実態と乖離した路線情報を運用で補正したいから。
US-013	管理者として, 駅マスタおよび駅↔路線の接続情報を画面から登録/修正/削除したい。なぜなら新駅開業/廃駅/接続変更を反映したいから。
US-014	ユーザとして, ヘッダ/フッタなど自分の識別を示す箇所には登録した氏名 (User.name) を表示してほしい。なぜならメールアドレスは ID 的な意味合いが強く、ログイン中アカウントを直感的に確認したい場面では氏名のほうが自然なため。
US-015	ユーザとして, プロフィール設定画面 (S08) のパスワード変更欄および削除確認用パスワード欄に, 画面表示時にブラウザの自動補完で値が埋まっていないでほしい。なぜなら入力していないパスワードが表示されているのは違和感があり, 誤操作 (そのまま「変更する」「削除する」を押下) の温床にもなるため。
US-016	ユーザとして, 経路登録/編集 (S04 / S06) の各区間で「種別」「路線」を選択した状態で「駅選択」ボタンを押下した時, 駅マスタ参照 (S07) が選択済み条件で絞り込んだ検索結果を最初から表示している状態で開いてほしい。なぜなら毎回同じ条件を popup 側で入力し直すのが煩わしいため。
US-017	ユーザとして, 駅マスタ参照 (S07) の検索フォームで種別を選択した時に路線リストの選択肢がその種別の路線のみに絞られ, 逆に路線を選択した時に種別もその路線が属する種別に自動で揃うようにしてほしい。なぜなら矛盾した条件 (例: 種別=バスで地下鉄路線) で空振り検索することを防ぐため。
US-018	システムとして, JR が運営する路線の表示名は「JR○○線」「JR○○本線」のように "JR" プレフィックスを付けて表示したい。なぜなら Wikidata 由来の素の名称 (中央本線, 関西本線, 等) では他事業者線と区別しづらいため。
US-019	ユーザとして, ログイン中の自分の氏名 (User.name) をログイン/サインアップ画面以外のすべての画面でヘッダ右上に常時表示してほしい。なぜなら現在ログインしているアカウントを画面に関わらず即座に確認したいから。「ユーザー：」のラベルは不要で氏名のみ表示し、クリックでアカウント設定 (/account) に遷移することで US-014 の動線を維持する。
